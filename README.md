# WHILE Language Parser

> C++ Recursive Descendent Parser for the **WHILE language**, that is a educational-purpose programming language.

**Attention:** I haven't followed any particular formal specification for the syntax, I've taken inspiration from different definitions of the language, but I changed it to match my personal needs.

## Structure

It is made of two components:

- **Lexer** whose task is to lexically analize the program and _tokenize_ the input;
- **Parser** who follows the _syntax rules_ of the language to validate it and produce an _AST_ (_Abstract Syntax Tree_) of the program.

### Lexer

The **lexer** component performs lexical analysis of the input program; that means that it doesn't catch syntax rules like `a + b`, but blindly _tokenizes_ the input recognizing the different types of tokens.

It can process both programs specified as **istream**, that is as a normal string, or as a program with `.wh` extension. 

It has a method `nextToken()` that returns a `Token` object composed of:

- `TokenType m_token_type` -> the type of the token (`NUMBER`, `IDENTIFIER`, `ASSIGN`, etc.);
- `std::string m_value` -> the actual string of the corresponding type;

### Parser
The parser is a top-down parser, in particular a **LL(1) Recursive Descending Parser**. It consumes token from the lexer and tries to match syntax rules, expressed in the grammar as *productions*. Doing this it creates an **Abstract Syntax Tree** (AST) that represents the program in a logical, non-strictrly concrete way: meaning that a node in the AST doesn't exactly match the concrete version generated by methodically applying the grammar rules.

#### Grammar
The parser abides to the following grammar, specified in EBNF

```
Program      ::= Statement*

Statement    ::= Assignment | Skip | IfStmt | WhileStmt

Assignment   ::= identifier ':=' Expr ';'
Skip         ::= 'skip'
IfStmt       ::= 'if' Pred 'then' Statement 'else' Statement 'endif'
WhileStmt    ::= 'while' Pred 'do' Statement 'endwhile'

Pred         ::= AndPred ( 'or' AndPred )*
AndPred      ::= UnaryPred ( 'and' UnaryPred )*
UnaryPred    ::= 'not' UnaryPred 
               | PrimaryPred

PrimaryPred  ::= 'true' 
               | 'false' 
               | '(' Pred ')' 
               | Relational

Relational   ::= Expr RelOp Expr
RelOp        ::= '<' | '<=' | '=' | '>' | '>='

Expr         ::= Term ( AddOp Term )*
Term         ::= Factor ( MulOp Factor )*
Factor       ::= '(' Expr ')' 
               | identifier 
               | number

AddOp        ::= '+' | '-'
MulOp        ::= '*' | '/'
```
Not that some precedence rules have been imposed like:
- `and` has an higher precedence than `or`
- `*` and `/` have higher precedences than `+` and `-`


## Build the project
The project is very easy to build, it uses **make** and it can build *lexer* and *parser* indipendently. In particular, for each of them 2 build configuration are provided:
- `make <name>` -> compiles the component specified (`lexer` or `parser`) and puts the executable in the `./bin` folder 
- `make test_<name>` -> compiles the tests specified module and puts the executable in the `./tests/bin` folder

Obviously the executable will have the name of the *make target*.
